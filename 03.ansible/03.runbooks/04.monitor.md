# Monitor

**스택**
- Keepalived: VIP (172.16.30.90)
- Prometheus: 노드 메트릭 수집 (Pull)
- Loki: 중앙 로그 저장소 (Push 수신)
- Grafana: 메트릭 및 로그 시각화 웹 콘솔
- Promtail: 각 VM의 시스템 로그 수집기 (클라이언트)

**모니터링 아키텍처 흐름**

```text
  [ Target VM (K8s / 기타 서버) ]         [ Monitor Server (VIP: 172.16.30.90) ]
  +--------------------------------+       +-----------------------------------------+
  |  Promtail (Agent)              | ----> |  Loki       (Port: 3100)  Push 수신     |
  |  Node Exporter (Port: 9100)    | <---- |  Prometheus (Port: 9090)  Pull 수집     |
  |                                |       |  Grafana    (Port: 3000)  시각화        |
  +--------------------------------+       +-----------------------------------------+
```

## 모니터링 구축 (설치)

> 최초 1회 실행

**배포**
```bash
# Terraform VM 생성
bash ~/workspace/terraform/02-run.sh prod apply services monitor-01

# Ansible 서비스 구성
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook \
  -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/monitor.yml
```

**검증**
```bash
# Ansible 접속 확인
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible -i ~/workspace/ansible/inventories/prod/hosts \
  monitoring_nodes -m ping

# 직접 접속 확인
ssh kosa@172.16.30.91

# VIP 보유 확인 (모니터링 노드에서 실행)
systemctl status keepalived
ip addr show | grep 172.16.30.90
# inet 172.16.30.90/32 scope global eth0

# Docker Compose 스택 상태 확인 (모니터링 노드에서 실행)
cd /opt/monitoring && sudo docker compose ps
# NAME        IMAGE                    STATUS
# prometheus  prom/prometheus:latest   Up   0.0.0.0:9090->9090/tcp
# loki        grafana/loki:latest      Up   0.0.0.0:3100->3100/tcp
# grafana     grafana/grafana:latest   Up   0.0.0.0:3000->3000/tcp
```

**서비스 확인**
> 컨트롤 노드 또는 VLAN 30 내 임의 호스트에서 실행

```bash
# VIP 경유 각 서비스 포트 응답 확인
curl -I http://172.16.30.90:3000        # Grafana
curl -I http://172.16.30.90:3100/ready  # Loki
curl -I http://172.16.30.90:9090        # Prometheus
```

**클라이언트 VM 검증**
> 컨트롤 노드에서 실행 — 전체 서버 일괄 확인

```bash
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible -i ~/workspace/ansible/inventories/prod/hosts \
  all -m shell -a "systemctl is-active promtail wazuh-agent prometheus-node-exporter" \
  --become
```

> 서비스가 없는 서버는 `inactive` 또는 `unknown`으로 표시됨

> 개별 VM 내부에서 상세 확인 시

```bash
# Promtail 기동 상태 및 포트(9080) 리슨 확인
systemctl status promtail
ss -tlnp | grep 9080
curl -I http://localhost:9080/targets

# Node Exporter 기동 상태 및 포트(9100) 리슨 확인
systemctl status prometheus-node-exporter
ss -tlnp | grep 9100
curl http://localhost:9100/metrics | head -n 20
```


## 모니터링 운영

> **원칙**: 스택 설정 및 Prometheus 스크랩 대상은 IaC 상태 유지를 위해 반드시 Ansible 변수 파일과 템플릿을 통해 관리함

**Grafana 비밀번호 변경**

**1. 변수 파일 수정 (컨트롤 노드)**
```bash
vi ~/workspace/ansible/inventories/prod/group_vars/monitoring_nodes.yml
```
```yaml
grafana_admin_password: "newpassword"
```

**2. 플레이북 실행 (상태 동기화)**
```bash
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/monitor.yml
```

**Prometheus 스크랩 대상 추가**

**1. prometheus 템플릿 수정 (컨트롤 노드)**
```bash
vi ~/workspace/ansible/roles/monitor/templates/prometheus.yml.j2
```
> `kubernetes-nodes` 스크랩 대상은 인벤토리의 `k8s_cluster` 그룹을 기반으로 자동 생성됨
> 추가 스크랩 대상(일반 VM 등)은 `static_configs`에 직접 항목 추가

**2. 플레이북 실행 (상태 동기화)**
```bash
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/monitor.yml
```

**수동 로그 전송 테스트**
- 신규 VM 추가 시 Loki 연결 정상 여부를 즉시 확인할 때 사용


## 트러블슈팅

**VIP가 할당되지 않음**
- **증상**: `ip addr show`에 172.16.30.90 미출력, curl 응답 없음
- **원인**: keepalived 미설치 또는 서비스 중단
- **해결**: `systemctl status keepalived` 확인 후 재시작; 미설치 시 playbook 재실행

**Docker 컨테이너 비정상 재시작**
- **증상**: `docker compose ps` 조회 시 특정 컨테이너 상태가 `Exit` 또는 Restarting
- **원인**: `/opt/monitoring/` 하위 디렉토리 권한 문제 또는 설정 파일 포맷 오류
- **해결**:
```bash
# 컨테이너 로그 확인
sudo docker compose -f /opt/monitoring/docker-compose.yml logs loki

# Grafana 디렉토리 권한 수정 (UID 472)
sudo chown -R 472:472 /opt/monitoring/grafana
```
