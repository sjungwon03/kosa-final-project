# DNS

**스택**
- Keepalived: VIP
- CoreDNS: etcd 기반 내부 도메인 서비스
- etcd: DNS 레코드 저장소

## DNS 구축 (설치)

> 최초 1회 실행

**배포**
```bash
# Terraform VM 생성
bash ~/workspace/terraform/02-run.sh prod apply dns

# Ansible 서비스 구성
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook \
  -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/dns.yml
```

**검증**
```bash
# Ansible 접속 확인
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible -i ~/workspace/ansible/inventories/prod/hosts \
  dns_servers -m ping

# 직접 접속 확인
ssh kosa@172.16.30.11
ssh kosa@172.16.30.12

# 두 노드에서 각각 실행
systemctl status etcd coredns keepalived

# etcd 클러스터 헬스 및 리더 확인 (IS LEADER 컬럼으로 구분)
ETCDCTL_API=3 etcdctl \
  --endpoints=http://172.16.30.11:2379,http://172.16.30.12:2379 \
  endpoint health

ETCDCTL_API=3 etcdctl \
  --endpoints=http://172.16.30.11:2379,http://172.16.30.12:2379 \
  endpoint status --write-out=table

# VIP 보유 노드 확인
ip addr show | grep 172.16.30.10
# inet 172.16.30.10/32 scope global eth0
```

**DNS 확인**
> 컨트롤 노드 또는 VLAN 30 내 임의 호스트에서 실행

```bash
# VIP 경유 쿼리
dig @172.16.30.10 nexus.mgmt.local +short
dig @172.16.30.10 dns.svc.local +short
dig @172.16.30.10 vault.sec.local +short

# 인터넷 포워딩 확인 (pfSense 경유)
dig @172.16.30.10 google.com +short

# etcd 전체 레코드 조회
ETCDCTL_API=3 etcdctl --endpoints=http://172.16.30.11:2379 \
  get /coredns --prefix --keys-only
```

**Failover 테스트**
```bash
# MASTER(dns-01)에서 CoreDNS 중단
sudo systemctl stop coredns

# VIP 유지 및 쿼리 응답 확인 (컨트롤 노드에서)
ping -c 3 172.16.30.10
dig @172.16.30.10 nexus.mgmt.local +short

# 복구
sudo systemctl start coredns
```


## DNS 레코드 관리 (운영)

> **원칙**: 모든 DNS 레코드는 IaC 상태 유지를 위해 반드시 Ansible 변수 파일을 통해 관리함

**레코드 추가 및 반영**

**1. 변수 파일 수정 (컨트롤 노드)**
```bash
vi ~/workspace/ansible/inventories/prod/group_vars/dns_servers.yml
```
```yaml
# dns_records 리스트에 대상 추가/수정/삭제
  - { name: "newservice", ip: "172.16.30.xx", domain: "mgmt.local" }
```

**2. 플레이북 실행 (상태 동기화)**
```bash
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/dns.yml
```
> etcd 레코드 등록 태스크는 멱등성이 보장되므로 플레이북을 다시 돌려도 기존 데이터에 영향을 주지 않음

**긴급 수동 제어**
- 장애 상황 등 불가피하게 수동 제어가 필요한 경우 사용
- 조치 완료 후 반드시 Ansible 변수 파일에 동기화해야 함

```bash
# 추가 — 형식: /coredns/<도메인 역순>/<호스트명>
ETCDCTL_API=3 etcdctl --endpoints=http://172.16.30.11:2379 \
  put /coredns/local/mgmt/newservice '{"host":"172.16.30.xx"}'

# 삭제
ETCDCTL_API=3 etcdctl --endpoints=http://172.16.30.11:2379 \
  del /coredns/local/mgmt/newservice
```


## 트러블슈팅

**CoreDNS 53번 포트 충돌**
- **증상**: CoreDNS 서비스 시작 실패 (`address already in use`)
- **원인**: Ubuntu 기본 프로세스(`systemd-resolved`)가 53번 포트를 먼저 선점함
- **해결**: 앤서블 플레이북에서 `no-stub.conf`를 배포하여 기본 프로세스를 끄도록 자동 조치함

**apt 패키지 잠금 (dpkg lock) 에러**
- **증상**: 앤서블 패키지 설치 태스크 중 에러 발생
- **원인**: VM 생성 직후 Ubuntu 백그라운드 무인 업데이트(unattended-upgrades)가 동작하여 apt를 잠금(lock)
- **해결**: 플레이북을 재실행하거나, 앤서블이 lock 해제를 기다리도록 조치함

**서비스 쿼리 무응답 (VIP 문제)**
- **증상**: VIP(`172.16.30.10`) 경유 접근 시 무응답
- **원인**: 노드 다운 또는 Keepalived 데몬 중단으로 VIP가 할당되지 않음
- **확인**: 각 노드에서 `ip addr show` 및 `systemctl status keepalived` 확인
