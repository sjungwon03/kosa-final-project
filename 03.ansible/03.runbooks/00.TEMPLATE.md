## [서비스명] 구축 (설치)

> 최초 1회 실행

**배포**
```bash
# Terraform VM 생성
bash ~/workspace/terraform/02-run.sh prod apply [role]

# Ansible 서비스 구성
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook \
  -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/[playbook].yml
```

**검증**
```bash
# Ansible 접속 확인
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible -i ~/workspace/ansible/inventories/prod/hosts \
  [host_group] -m ping

# 직접 접속 확인
ssh kosa@[노드 IP]

# 서비스 상태 확인
systemctl status [서비스명]
```

**서비스 확인**
> 컨트롤 노드 또는 VLAN 30 내 임의 호스트에서 실행

```bash
# 포트 응답 확인
curl -I http://[서비스 IP 또는 VIP]:[PORT]
```

**Failover 테스트**
> VIP(Keepalived) 구성 서비스에만 해당

```bash
# MASTER 노드에서 서비스 중단
sudo systemctl stop [서비스명]

# VIP 이관 및 응답 유지 확인 (컨트롤 노드에서)
curl -I http://[VIP]:[PORT]

# 복구
sudo systemctl start [서비스명]
```
