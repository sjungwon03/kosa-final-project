# Nexus


폐쇄망 전환 전 Nexus 준비
pfSense VLAN 30 룰 변경(폐쇄망 전환) 전에 Nexus에 미러링/업로드할 항목 체크리스트
빠진 항목이 있으면 폐쇄망 이후 해당 서비스 설치 불가

## 전환 절차

1. 위 항목 전부 Nexus 업로드/캐싱 완료 확인
2. pfSense VLAN 30 룰 변경
   - allow-all 제거
   - 허용: VLAN 30 내부 상호 통신, 관리망(192.168.34.x) SSH, Nexus 접근, K8s 포트
3. 전환 후 임의 노드에서 패키지 설치 테스트
   ```bash
   apt-get install -y curl   # Nexus proxy 경유 확인
   ```
