# HAProxy

**스택**
- HAProxy: L4/L7 로드밸런서
- Keepalived: VIP 관리 (172.16.20.25)

**구성**
- HAProxy VIP: 172.16.20.25 (haproxy.svc.local)
- haproxy-01: 172.16.20.26 (MASTER)
- haproxy-02: 172.16.20.27 (BACKUP)

## HAProxy 구축 (설치)

> 최초 1회 실행

**배포**
```bash
# [컨트롤 노드] Terraform VM 생성
bash ~/workspace/terraform/02-run.sh prod apply haproxy

# [컨트롤 노드] Ansible 서비스 구성
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook \
  -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/haproxy.yml
```

**검증**
```bash
# Keepalived 상태 확인
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible loadbalancers \
  -i ~/workspace/ansible/inventories/prod/hosts \
  -m shell -a "systemctl is-active keepalived haproxy" --become

# VIP 할당 확인 (MASTER 노드에만 172.16.20.25 보여야 함)
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible loadbalancers \
  -i ~/workspace/ansible/inventories/prod/hosts \
  -m shell -a "ip addr show | grep 172.16.20.25" --become

# HAProxy 응답 확인
curl -I http://172.16.20.25
```

**Failover 테스트**
```bash
# MASTER(haproxy-01)에서 서비스 중단
sudo systemctl stop keepalived

# VIP 이관 확인 (컨트롤 노드에서)
curl -I http://172.16.20.25

# 복구
sudo systemctl start keepalived
```
