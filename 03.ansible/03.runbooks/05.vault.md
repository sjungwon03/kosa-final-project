# Vault

**스택**
- HashiCorp Vault: 시크릿 관리, PKI 인증서 발급
- Raft (통합 스토리지): 별도 외부 스토리지 없이 노드 간 합의

**구성**
- vault-01: 172.16.30.21 (최초 init 대상)
- vault-02: 172.16.30.22
- API 포트: 8200 / 클러스터 포트: 8201
- TLS: 현재 비활성화 (TODO: Vault PKI로 발급 후 활성화)

> `site.yml`에서 vault.yml은 주석 처리 상태 — 반드시 단독으로 실행


## Vault 구축 (설치)

> 최초 1회 실행

**배포**
```bash
# Terraform VM 생성
bash ~/workspace/terraform/02-run.sh prod apply vault

# Ansible 서비스 구성
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook \
  -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/vault.yml
```

**검증**
```bash
# Ansible 접속 확인
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible -i ~/workspace/ansible/inventories/prod/hosts \
  vault -m ping

# 직접 접속 확인
ssh kosa@172.16.30.21
ssh kosa@172.16.30.22

# 각 노드에서 실행
systemctl status vault
export VAULT_ADDR="http://127.0.0.1:8200"
vault status
# Key                Value
# Initialized        false   ← 초기화 전 정상
# Sealed             true
```

**초기화 (Init)**
> **vault-01에서만 1회 실행** — 결과(Unseal Key, Root Token)는 반드시 안전한 곳에 저장

```bash
# vault-01에서 실행
export VAULT_ADDR="http://127.0.0.1:8200"
vault operator init

# 출력 예시
# Unseal Key 1: xxxx
# Unseal Key 2: xxxx
# Unseal Key 3: xxxx
# Unseal Key 4: xxxx
# Unseal Key 5: xxxx
# Initial Root Token: hvs.xxxx
```

> 키 5개 중 3개가 있어야 Unseal 가능 (Shamir 비밀 분산)
> Root Token은 초기 설정 후 폐기 권장, 이후 별도 정책 기반 토큰 사용

**Unseal**
> init 완료 후 **각 노드에서** 실행 — Unseal Key 3개 입력

```bash
export VAULT_ADDR="http://127.0.0.1:8200"

vault operator unseal  # Key 1 입력
vault operator unseal  # Key 2 입력
vault operator unseal  # Key 3 입력

# 완료 후 확인
vault status
# Initialized    true
# Sealed         false  ← Unseal 완료
# HA Enabled     true
```

**Raft 클러스터 확인**
> vault-01 (Active 노드)에서 실행

```bash
export VAULT_ADDR="http://127.0.0.1:8200"
vault login  # Root Token 입력

# 클러스터 멤버 확인
vault operator raft list-peers
# Node       Address                    State       Voter
# vault-01   172.16.30.21:8201          leader      true
# vault-02   172.16.30.22:8201          follower    true
```


## Vault 운영

> **원칙**: Vault 재시작 시 항상 Sealed 상태로 돌아오므로 Unseal 수동 수행 필요

**재시작 후 Unseal**
```bash
# 각 노드에서 실행
export VAULT_ADDR="http://127.0.0.1:8200"
vault operator unseal  # Key 1
vault operator unseal  # Key 2
vault operator unseal  # Key 3
```

**Active 노드 확인**
```bash
# 어느 노드에서든 실행
export VAULT_ADDR="http://127.0.0.1:8200"
vault status | grep -E "HA Mode|Active Node"
```

**설정 변경 및 재배포**

**1. 변수 파일 수정 (컨트롤 노드)**
```bash
vi ~/workspace/ansible/inventories/prod/group_vars/vault.yml
# 또는 roles/vault/defaults/main.yml
```

**2. 플레이북 실행 (상태 동기화)**
```bash
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/vault.yml
```
> 설정 파일 변경 시 handler가 vault 재시작 — Unseal 필요


## 트러블슈팅

**Vault가 Sealed 상태**
- **증상**: `vault status`에서 `Sealed: true`, API 요청에 503 응답
- **원인**: 서비스 재시작 또는 노드 재부팅 시 항상 Sealed 상태로 초기화됨
- **해결**: 각 노드에서 `vault operator unseal` 3회 실행

**Raft 클러스터에 노드가 없음**
- **증상**: `vault operator raft list-peers` 결과에 follower 노드 미표시
- **원인**: follower 노드의 vault 서비스 미기동 또는 8201 포트 통신 차단
- **해결**: follower 노드에서 `systemctl status vault` 확인 후 재시작; Unseal 필요

**apt 패키지 잠금 (dpkg lock) 에러**
- **증상**: Ansible 패키지 설치 태스크 중 에러 발생
- **원인**: VM 생성 직후 Ubuntu 무인 업데이트(unattended-upgrades)가 apt를 잠금(lock)
- **해결**: 플레이북 재실행 (common role이 lock 해제 대기 처리함)

**Init 결과 분실**
- **증상**: Unseal Key 또는 Root Token 분실
- **원인**: 초기화 시 출력 결과를 저장하지 않음
- **해결**: 복구 불가 — `vault operator init` 재실행 불가, Vault 데이터 초기화 후 재구축 필요
