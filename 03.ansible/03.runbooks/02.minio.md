# MinIO

**스택**
- MinIO: S3 호환 오브젝트 스토리지
- mc: MinIO 클라이언트 CLI

**구성**
- minio-01
- API 포트: 9000 / 콘솔 포트: 9001
- 도메인: minio.mgmt.local
- 기본 버킷: `terraform-state` (Terraform backend)

## MinIO 구축 (설치)

> 최초 1회 실행

**배포**
```bash
bash ~/workspace/terraform/02-run.sh prod apply services minio-01

ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook \
  -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/minio.yml
```

**검증**
```bash
# Ansible 접속 확인
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible -i ~/workspace/ansible/inventories/prod/hosts \
  minio -m ping

# 직접 접속 확인
ssh kosa@172.16.30.70

# 서비스 상태 확인 (minio-01에서 실행)
systemctl status minio

# API 응답 확인
curl -I http://172.16.30.70:9000/minio/health/live

# 버킷 목록 확인
mc alias set local http://127.0.0.1:9000 kosa kosa1004
mc ls local
# [2026-05-18 16:16:33 KST]     0B terraform-state/
```

**콘솔 접속**
> 브라우저에서 http://172.16.30.70:9001 접속
> 루트 계정: `kosa`(추가 사용자 생성 필요 시 콘솔 > Identity > Users)
> 콘솔에서 확인 가능한 항목: 버킷 목록, 서비스별 state 파일, 오브젝트 크기 및 수정 시각


## Terraform Backend 전환

> MinIO 구축 완료 후 1회 수행
> 컨트롤 노드에 기존 배포 이력(state)이 있는 경우 `migrate-state`로 이전
> 신규 환경(state 없음)은 `terraform init`만 실행하면 됨

**1. env/prod/main.tf backend 블록 확인**
> 이미 코드에 포함되어 있음 — 내용만 확인하고 다음 단계로 진행
```hcl
terraform {
  required_providers {
    proxmox = {
      source  = "bpg/proxmox"
      version = "~> 0.66"
    }
  }

  backend "s3" {
    # key는 02-run.sh에서 -backend-config로 주입 (서비스별 state 분리)
    bucket = "terraform-state"
    region = "us-east-1"

    endpoints = {
      s3 = "http://172.16.30.70:9000"
    }
    access_key                  = "kosa"
    secret_key                  = "kosa1004"

    skip_credentials_validation  = true
    skip_metadata_api_check      = true
    skip_region_validation       = true
    skip_requesting_account_id   = true
    use_path_style               = true
  }
}
```

**2. 컨트롤 노드에 배포**
```bash
bash 03.ansible/03-deploy-to-control.sh
```

**3. 기존 state를 MinIO로 이전 (컨트롤 노드)**
> `{ROLE}`은 이전할 서비스명으로 교체 (예: k8s, minio, vault)
```bash
cd ~/workspace/terraform/env/prod
terraform init -migrate-state -backend-config="key=prod/{ROLE}/terraform.tfstate"
```

> 이후부터 state는 MinIO `terraform-state` 버킷에 서비스별로 저장됨
> 기존에 배포된 VM(K8s 등)이 있어도 state 내용은 그대로 이전되므로 안전하게 수행 가능


## Terraform State 분리

> 서비스별로 배포를 독립적으로 수행하므로 state도 서비스별로 분리 관리

**구조**
```
terraform-state/
└── prod/
    ├── k8s/terraform.tfstate
    ├── minio/terraform.tfstate
    ├── vault/terraform.tfstate
    └── ...
```

**동작 방식**
- `02-run.sh`가 init 시 `-backend-config="key=${ENV}/${ROLE}/terraform.tfstate"` 자동 주입
- `02-run.sh prod apply k8s` → `prod/k8s/terraform.tfstate` 사용
- main.tf에 key 하드코딩 없음

**state 파일 확인**
```bash
# [minio-01에서 실행]
mc ls local/terraform-state/prod/
# prod/k8s/terraform.tfstate
# prod/minio/terraform.tfstate
```

또는 콘솔(http://172.16.30.70:9001) > `terraform-state` 버킷 > `prod/` 경로에서 확인


## MinIO 운영

> **원칙**: MinIO 설정은 IaC 상태 유지를 위해 반드시 Ansible 변수 파일을 통해 관리함

**1. 비밀번호 변경 (컨트롤 노드)**
```bash
# 변수 파일 수정
vi ~/workspace/ansible/roles/minio/defaults/main.yml
```

main.yml
```yaml
minio_root_user: "kosa"
minio_root_password: ""
```

**2. 플레이북 실행 (상태 동기화)**
```bash
ANSIBLE_CONFIG=~/workspace/ansible/ansible.cfg \
  ansible-playbook -i ~/workspace/ansible/inventories/prod/hosts \
  ~/workspace/ansible/playbooks/minio.yml
```
> env/prod/main.tf의 `access_key` / `secret_key`도 동일하게 수정 필요


## 트러블슈팅

**MinIO 서비스 미기동**
- **증상**: `systemctl status minio` 실패 또는 포트 9000 응답 없음
- **원인**: 바이너리 다운로드 실패 (인터넷 미연결) 또는 데이터 디렉토리 권한 문제
- **해결**:
```bash
# 바이너리 존재 확인
ls -la /usr/local/bin/minio

# 데이터 디렉토리 권한 확인
ls -la /data/minio

# 서비스 로그 확인
journalctl -u minio -n 50
```

**terraform init 시 AWS account ID 오류**
- **증상**: `Retrieving AWS account details: AWS account ID not previously found` / `InvalidClientTokenId`
- **원인**: backend 블록에 `skip_requesting_account_id = true` 누락, 또는 `endpoint` / `force_path_style` 파라미터가 deprecated된 구버전 형식 사용
- **해결**: 아래 파라미터로 교체
```hcl
endpoints = { s3 = "http://..." }   # endpoint 대체
use_path_style = true               # force_path_style 대체
skip_requesting_account_id = true   # 추가
```

**mc alias set 실행 시 connection refused**
- **증상**: `dial tcp 127.0.0.1:9000: connect: connection refused`
- **원인**: systemd가 minio 서비스를 기동했으나 포트가 준비되기 전에 mc 명령이 실행됨 (race condition)
- **해결**: 플레이북 재실행 (minio role에 `wait_for` 태스크로 자동 대기 처리됨)
