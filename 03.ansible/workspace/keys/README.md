# SSH Public Keys

인프라 접근 제어를 위한 사용자별 SSH 공개키 관리

## 용도

1. **VM 초기 프로비저닝**: Terraform 실행 시 `user_account`에 주입되어 생성된 모든 가상 서버의 `kosa` 계정 접속 권한 부여
2. **배포 제어**: 컨트롤 노드 및 각 환경별 서버에 대한 팀원별 개별 접속 허용 및 Ansible 자동화 연동

## 관리 가이드

| 파일명 | 내용 | 관리 기준 |
|---|---|---|
| `*.pub` | 사용자의 SSH 공개키 | **Git 커밋 대상**. 팀원 추가 시 이곳에 파일 업로드 |
| (확장자 없음) | 사용자의 SSH 비밀키 | **절대 커밋 금지**. 개인 로컬에서만 보관 |

---

> [!TIP]
> 공개키(Public Key)는 서버의 `authorized_keys`에 등록되는 자물쇠 역할을 하므로 외부에 노출되어도 안전합니다. 팀 프로젝트에서는 이곳에 공개키를 모아두어 인프라 생성과 동시에 접근 권한을 자동화합니다.
