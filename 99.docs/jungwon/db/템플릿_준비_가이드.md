# Proxmox VM Template 준비 가이드

## 1. 템플릿 개요

### 템플릿 기반 VM 생성

**작동 방식**:
- Proxmox에서 템플릿 VM을 미리 생성
- Terraform이 템플릿을 클론하여 새 VM 생성
- Cloud-init으로 VM 초기화 (IP, SSH Key 등)

**장점**:
- 빠른 VM 생성 (전체 설치 불필요)
- 일관된 환경 (모든 VM이 동일한 기본 설정)
- 재사용 가능 (여러 클러스터에서 사용)

## 2. Ubuntu 22.04 템플릿 생성 과정

### Step 1: Ubuntu VM 생성

**Proxmox GUI**:
```
1. VM 생성
   ├── Node 선택 (예: pve1)
   ├── VM ID: 9001 (템플릿용 ID)
   ├── Name: ubuntu-22.04-template
   └── OS Type: Linux
   
2. System 설정
   ├── Graphic card: Default
   ├── Machine: q35 (또는 i440fx)
   ├── SCSI Controller: VirtIO SCSI
   ├── Qemu Agent: Enable
   └── BIOS: SeaBIOS
   
3. Disk 설정
   ├── Bus: VirtIO
   ├── Storage: rbd-storage (Ceph RBD)
   ├── Disk size: 20G
   └── SSD Emulation: Enable
   
4. CPU 설정
   ├── Cores: 2
   ├── Type: host (성능 최적화)
   
5. Memory 설정
   ├── Memory: 2048 MiB
   ├── Ballooning: Disable (템플릿은 고정)
   
6. Network 설정
   ├── Model: VirtIO (best performance)
   ├── Bridge: vmbr1
   └── VLAN Tag: 0 (또는 사용 VLAN)
   
7. Cloud-init 설정 (중요!)
   ├── OS Type: Linux
   ├── CIuser: ubuntu
   ├── CIpassword: (설정하거나 SSH Key만 사용)
   ├── SSH Public Key: 입력
   ├── IP config: DHCP (템플릿은 DHCP 사용)
   └── DNS Servers: 8.8.8.8, 8.8.4.4
```

### Step 2: Ubuntu ISO 부팅

**ISO 준비**:
```bash
# Ubuntu 22.04 Server ISO 다운로드
wget https://releases.ubuntu.com/22.04/ubuntu-22.04-live-server-amd64.iso

# Proxmox에 ISO 업로드
# GUI: pve1 → local (storage) → ISO Images → Upload
# 또는 CLI:
scp ubuntu-22.04-live-server-amd64.iso root@pve1:/var/lib/vz/template/iso/
```

**VM 부팅**:
```
1. VM (9000) 선택
2. Hardware → Add → CD/DVD Drive
   ├── Bus: IDE
   └── ISO: ubuntu-22.04-live-server-amd64.iso
   
3. VM Start
4. Console 열기 (noVNC 또는 SPICE)
```

### Step 3: Ubuntu 설치

**설치 과정**:
```
1. Language: English
   
2. Installer update available: Continue without updating
   
3. Keyboard configuration: Korean (또는 English)
   
4. Choose type of install: Ubuntu Server (default)
   
5. Network connections: 
   └── eth0: DHCP (템플릿은 DHCP 사용)
   
6. Configure proxy: (empty)
   
7. Configure Ubuntu archive mirror: Default
   
8. Guided storage configuration:
   ├── Use an entire disk
   └── Set up this disk as an LVM group: Yes
   
9. Storage configuration:
   └── Done
   
10. Confirm destructive action: Continue
   
11. Profile setup:
    ├── Your name: ubuntu
    ├── Your server's name: ubuntu-template
    ├── Pick a username: ubuntu
    ├── Choose a password: ubuntu (또는 복잡한 비밀번호)
    └── Confirm your password: ubuntu
    
12. SSH Setup:
    └── Install OpenSSH server: Yes
    └── Import SSH identity: No
    
13. Featured Server Snaps:
    └── Skip (추후 Ansible로 설치)
    
14. Installation complete:
    └── Reboot Now

    ├── VM Stop
    ├── Hardware → CD/DVD Drive → Remove

```

### Step 4: VM 초기 설정

**SSH 접속**:
```bash
# VM 부팅 완료 후 SSH 접속
ssh ubuntu@<VM_IP>
# Password: ubuntu
```

**기본 패키지 설치**:
```bash
# Update
sudo apt update && sudo apt upgrade -y

# 필수 패키지
sudo apt install -y \
    qemu-guest-agent \
    cloud-init \
    netplan.io \
    openssh-server \
    vim \
    wget \
    curl \
    jq \
    ca-certificates \
    gnupg \
    apt-transport-https \
    software-properties-common

# Qemu Guest Agent 설정
sudo systemctl enable qemu-guest-agent
sudo systemctl start qemu-guest-agent
```

**Cloud-init 설정**:
```bash
# Cloud-init 설정 파일 확인
cat /etc/cloud/cloud.cfg

# Cloud-init 초기화 (템플릿용)
sudo cloud-init clean
sudo rm -rf /var/lib/cloud/

# Netplan 설정 (Cloud-init용)
sudo cat > /etc/netplan/00-installer-config.yaml <<EOF
network:
  ethernets:
    ens18:
      dhcp4: true
  version: 2
EOF

sudo netplan apply
```

**SSH 설정**:
```bash
# SSH 설정
sudo sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
sudo sed -i 's/#PasswordAuthentication yes/PasswordAuthentication yes/' /etc/ssh/sshd_config

sudo systemctl restart sshd
```

**시스템 최적화**:
```bash
# Swap 비활성화 (템플릿용)
sudo swapoff -a
sudo sed -i '/swap/d' /etc/fstab

# 불필요한 파일 정리
sudo apt autoremove -y
sudo apt autoclean
sudo rm -rf /var/cache/apt/archives/*.deb

# 로그 정리
sudo journalctl --vacuum-time=1s
sudo rm -rf /var/log/*

# Machine-id 정리 (Cloud-init이 새로 생성)
sudo rm /etc/machine-id
sudo touch /etc/machine-id

# Hostname 정리
sudo hostnamectl set-hostname ubuntu-template
sudo sed -i 's/ubuntu-template/localhost/' /etc/hosts

# SSH Host Keys 정리 (VM 클론 시 새로 생성)
sudo rm -f /etc/ssh/ssh_host_*
```

### Step 5: 템플릿으로 변환

**VM 정지**:
```bash
# SSH에서
sudo poweroff

# 또는 Proxmox GUI
# VM (9000) → Stop
```

**템플릿 변환**:
```
1. Proxmox GUI
   ├── VM (9000) 선택
   ├── More → Convert to template
   └── Confirm
   
2. 또는 CLI
   qm template 9000
```

**템플릿 확인**:
```
# 템플릿 목록 확인
qm list

# 출력 예시:
VMID Name                 Status     Mem(MB)    ...
9000 ubuntu-22.04-template stopped    2048      (template)
```

## 3. Cloud-init 고급 설정

### 3-1. Cloud-init 구조

**Cloud-init 작동 방식**:
```
VM 시작:
  ├── Proxmox가 Cloud-init 데이터 생성
  │   ├── ISO 생성 (seed.iso)
  │   ├── network-config
  │   ├── user-data
  │   └── meta-data
  │
  ├── VM에 ISO 마운트
  │
  ├── Cloud-init이 ISO에서 설정 읽기
  │   ├── Network 설정 (IP, Gateway)
  │   ├── User 생성 (SSH Key)
  │   ├── Hostname 설정
  │   └── Package 설치
  │
  └── 설정 적용 완료
```

### 3-2. Proxmox Cloud-init 설정

**GUI 설정**:
```
VM → Cloud-init

User:
  ├── Username: ubuntu (또는 root)
  └── Password: (SSH Key만 사용하면 비워도 됨)
  
SSH Public Key:
  └── ssh-rsa AAAAB3Nza... (전체 키 입력)
  
Network:
  ├── IP Config (0):
  │   ├── IP: 172.16.20.20/24 (고정 IP)
  │   ├── Gateway: 172.16.20.1
  │
  ├── DNS Servers: 8.8.8.8, 8.8.4.4
  │
  └── Search domains: example.com
```

**CLI 설정**:
```bash
# Cloud-init 설정
qm set 100 \
  --ciuser ubuntu \
  --sshkeys "ssh-rsa AAAAB3Nza..." \
  --ipconfig0 "ip=172.16.20.20/24,gw=172.16.20.1" \
  --nameserver "8.8.8.8,8.8.4.4"
```

### 3-3. Terraform Cloud-init 설정

**terraform/modules/proxmox-vm/main.tf**:
```hcl
resource "proxmox_vm_qemu" "vm" {
  ...
  
  os_type  = "cloud-init"
  ciuser   = var.ciuser
  sshkeys  = var.ssh_public_key
  ipconfig0 = "ip=${var.ip_address},gw=${var.gateway}"
  
  ...
}
```

## 5. Ansible로 템플릿 준비

### 5-1. 템플릿 준비 플레이북

**ansible/playbooks/template-preparation.yml**:
```yaml
---
- name: Prepare Ubuntu VM as Template
  hosts: template_vm
  become: yes
  tasks:
    - name: Update apt cache
      apt:
        update_cache: yes
        cache_valid_time: 3600

    - name: Upgrade all packages
      apt:
        upgrade: dist

    - name: Install essential packages
      apt:
        name:
          - qemu-guest-agent
          - cloud-init
          - netplan.io
          - openssh-server
          - vim
          - wget
          - curl
          - jq
          - ca-certificates
          - gnupg
          - apt-transport-https
          - software-properties-common
        state: present

    - name: Enable Qemu Guest Agent
      systemd:
        name: qemu-guest-agent
        enabled: yes
        state: started

    - name: Configure SSH
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: "{{ item.regexp }}"
        line: "{{ item.line }}"
      loop:
        - { regexp: '#PermitRootLogin', line: 'PermitRootLogin yes' }
        - { regexp: '#PasswordAuthentication', line: 'PasswordAuthentication yes' }

    - name: Restart SSH
      systemd:
        name: sshd
        state: restarted

    - name: Disable swap
      command: swapoff -a
      when: ansible_swaptotal_mb > 0

    - name: Remove swap from fstab
      mount:
        name: swap
        fstype: swap
        state: absent

    - name: Clean apt cache
      apt:
        autoclean: yes

    - name: Remove unnecessary packages
      apt:
        autoremove: yes

    - name: Clean logs
      shell: |
        journalctl --vacuum-time=1s
        rm -rf /var/log/*

    - name: Reset machine-id
      file:
        path: /etc/machine-id
        state: absent

    - name: Create empty machine-id
      file:
        path: /etc/machine-id
        state: touch
        mode: '0644'

    - name: Clear hostname
      hostname:
        name: ubuntu-template

    - name: Update hosts file
      lineinfile:
        path: /etc/hosts
        regexp: '^127.0.1.1'
        line: '127.0.1.1 localhost'

    - name: Remove SSH host keys
      file:
        path: "{{ item }}"
        state: absent
      loop:
        - /etc/ssh/ssh_host_rsa_key
        - /etc/ssh/ssh_host_rsa_key.pub
        - /etc/ssh/ssh_host_ecdsa_key
        - /etc/ssh/ssh_host_ecdsa_key.pub
        - /etc/ssh/ssh_host_ed25519_key
        - /etc/ssh/ssh_host_ed25519_key.pub

    - name: Clean Cloud-init
      command: cloud-init clean
      args:
        removes: /var/lib/cloud

    - name: Remove Cloud-init data
      file:
        path: /var/lib/cloud
        state: absent

    - name: Shutdown VM
      command: shutdown -h now
      async: 0
      poll: 0
      ignore_errors: yes
```

### 5-2. 실행 방법

**템플릿 VM 인벤토리**:
```ini
[template_vm]
ubuntu-template ansible_host=<VM_IP>
```

**플레이북 실행**:
```bash
ansible-playbook -i inventory/template.ini playbooks/template-preparation.yml
```

## 6. Terraform에서 템플릿 사용

### 6-1. 템플릿 변수

**terraform.tfvars**:
```hcl
template_name = "ubuntu-22.04-template"
```

### 6-2. VM 클론 과정

**Terraform 작동 방식**:
```
1. Proxmox API로 템플릿 확인
   ├── qm list | grep ubuntu-22.04-template
   
2. 템플릿에서 Full Clone
   ├── qm clone 9000 <VMID> --name <VM_NAME>
   └── Full Clone (전체 복사, 독립 VM)
   
3. VM 설정 변경
   ├── CPU, Memory 설정
   ├── Disk 크기 변경 (필요시)
   └── Network 설정
   
4. Cloud-init 데이터 생성
   ├── Proxmox가 seed.iso 생성
   └── IP, SSH Key, User 설정
   
5. VM 시작
   ├── VM Boot
   └── Cloud-init 실행 (설정 적용)
   
6. SSH 접속 가능
   ├── Cloud-init 완료 후
   └── SSH Key로 접속
```

### 6-3. Full Clone vs Linked Clone

**Full Clone** (현재 설정):
```
장점:
  ├── 템플릿과 독립
  ├── 템플릿 삭제 가능
  └── 성능: 템플릿 영향 없음
  
단점:
  ├── 디스크 공간 많이 사용
  ├── 생성 시간 더 걸림
```

**Linked Clone**:
```
장점:
  ├── 디스크 공간 절약
  ├── 생성 빠름
  
단점:
  ├── 템플릿 필요
  ├── 템플릿 삭제 불가
```

**현재 Terraform 설정**:
```hcl
clone      = var.template_name
full_clone = true  # Full Clone 사용
```

## 8. 문제 해결

### Cloud-init 작동 안 함

**확인**:
```bash
# VM에서
sudo cloud-init status
# status: done

# 로그 확인
sudo cat /var/log/cloud-init-output.log
```

**해결**:
```bash
# Cloud-init 재실행
sudo cloud-init clean
sudo cloud-init init
```

### SSH 접속 불가

**확인**:
```bash
# VM에서
sudo systemctl status sshd
sudo cat /etc/ssh/sshd_config

# SSH Key 확인
sudo cat /home/ubuntu/.ssh/authorized_keys
```

**해결**:
```bash
# SSH Key 재설정 (Proxmox GUI)
# VM → Cloud-init → SSH Public Key 재입력
# VM Reboot
```

### Qemu Agent 작동 안 함

**확인**:
```bash
# VM에서
sudo systemctl status qemu-guest-agent

# Proxmox에서
qm agent <VMID> ping
```

**해결**:
```bash
# VM에서
sudo systemctl restart qemu-guest-agent
sudo systemctl enable qemu-guest-agent
```
