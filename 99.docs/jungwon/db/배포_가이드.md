## 2. 사전 준비

### 2-1. Proxmox 준비

**Proxmox 클러스터 (4 노드)**:
```
pve1, pve2, pve3, pve4

Requirements:
  ├── Proxmox VE 7.x or 8.x
  ├── Ceph cluster configured
  ├── rbd-storage pool created
  ├── Network: vmbr1 (VLAN-aware bridge)
  ├── HA Group: percona-ha (사전 생성)
  └── API Token for Terraform
```

**HA Group 생성 (사전 준비)**:
```bash
# HA Group 생성 (Proxmox 노드에서)
ha-manager add-group percona-ha \
  --nodes pve1,pve2,pve3,pve4 \
  --restricted 1

# 확인
ha-manager group list
ha-manager status
```

**Proxmox 설정 확인**:
```bash
# Proxmox 노드 확인
pvecm status

# Ceph 상태 확인
ceph status

# Storage 확인
pvesm status

# 출력 예시:
NAME        TYPE     STATUS  ACTIVE   CONTENT
rbd-storage rbd      active  yes      images,rootdir
```

### 2-2. pfSense 준비

**VLAN 설정**:
```
VLAN 20 (DMZ):
  ├── VLAN ID: 20
  ├── Gateway: 172.16.20.1
  ├── IP Range: 172.16.20.0/24
  └── Firewall: Allow MySQL to VLAN 30
  
VLAN 30 (Internal):
  ├── VLAN ID: 30
  ├── Gateway: 172.16.30.1
  ├── IP Range: 172.16.30.0/24
  └── Firewall: Block VLAN 20, Allow MySQL from VLAN 20
```

**pfSense 방화벽 규칙**:
```
VLAN 20 → VLAN 30:
  ├── Action: Pass
  ├── Protocol: TCP
  ├── Source: VLAN 20 (172.16.20.0/24)
  ├── Destination: VLAN 30 (172.16.30.10-12)
  └── Port: 3306 (MySQL)
  
VLAN 30 → VLAN 20:
  ├── Action: Block
  ├── Protocol: Any
  └── Purpose: Internal 네트워크 보호
```

### 2-3. Ubuntu Template 준비

**Template VM (ID: 9001)**:
```bash
# Ubuntu 22.04 VM 생성
qm create 9001 --name ubuntu-22.04-template \
  --cores 2 --memory 2048 \
  --net0 virtio,bridge=vmbr1 \
  --scsi0 rbd-storage:20
  
# Ubuntu 설치 후 Ansible로 템플릿 준비
ansible-playbook -i ansible/inventory/template.ini \
  ansible/playbooks/template-preparation.yml

# Template 변환
qm template 9001
```

## 3. Terraform 설정

### 3-1. terraform.tfvars 작성

```bash
cd terraform

# terraform.tfvars 생성
cp terraform.tfvars.example terraform.tfvars

# 값 수정
vim terraform.tfvars
```

**필수 설정값**:
```hcl
# Proxmox API
proxmox_api_url           = "https://pve1.example.com:8006/api2/json"
proxmox_api_token_id      = "root@pam!terraform"
proxmox_api_token_secret   = "실제-token-secret"

# SSH Key
ssh_public_key             = "ssh-rsa AAAA... 실제 SSH 키"

# Template
template_vmid              = 9001

# Account
ciuser                     = "kosa"
cipassword                 = "kosa1004"

# Storage
storage                    = "rbd-storage"

# Network
dmz_ip_prefix              = "172.16.20"
internal_ip_prefix         = "172.16.30"
dmz_gateway                = "172.16.20.1"
internal_gateway           = "172.16.30.1"

# HA Configuration
ha_enabled                 = true
ha_group                   = "percona-ha"
```

### 3-2. Terraform 실행

```bash
cd terraform

# 초기화
terraform init

# 계획 확인
terraform plan

# 출력:
# Plan: 7 to add, 0 to change, 0 to destroy.
# + module.percona_cluster.module.percona[0]
# + module.percona_cluster.module.haproxy[0]
# + module.percona_cluster.module.proxysql[0]
# ...

# 배포
terraform apply

# "yes" 입력

# Terraform 작동:
# 1. Template 9001에서 VM Full Clone
# 2. CPU/Memory/Disk 설정
# 3. VLAN 설정 (DMZ 20, Internal 30)
# 4. Static IP 설정 (Cloud-init)
# 5. User: kosa, Password: kosa1004
# 6. HA 설정: ha = "started" (HA 활성화)
# 7. Ansible inventory 생성

# HA 확인
ha-manager resource list
```

## 4. Ansible 실행

### 4-1. Ansible 배포

```bash
cd ansible

# SSH 연결 테스트
ansible -i inventory/hosts.ini all -m ping

# 출력:
pxc-prod-pxc-1 | SUCCESS
pxc-prod-pxc-2 | SUCCESS
pxc-prod-haproxy-1 | SUCCESS

# Main playbook 실행
ansible-playbook -i inventory/hosts.ini playbooks/site.yml

# Ansible 작동:
# 1. Common role: 모든 VM 기본 설정
# 2. HAProxy role: DMZ 로드밸런서 설정
# 3. ProxySQL role: DMZ MySQL 프록시 설정
# 4. Percona role (bootstrap): pxc-1 클러스터 초기화
# 5. Percona role: pxc-2, pxc-3 클러스터 가입
```

### 4-2. 배포 과정 상세

**Common Role**:
```
모든 VM:
  ├── apt update & upgrade
  ├── 기본 패키지 설치
  ├── hosts 파일 설정
  ├── timezone UTC
  └── SSH 설정
```

**HAProxy Role**:
```
haproxy-1, haproxy-2 (DMZ):
  ├── HAProxy 패키지 설치
  ├── Backend: Percona (172.16.30.10-12)
  ├── Port 3306: 읽기 분산
  ├── Port 3307: 쓰기集中
  └── Stats: Port 8404
```

**ProxySQL Role**:
```
proxysql-1, proxysql-2 (DMZ):
  ├── ProxySQL 패키지 설치
  ├── MySQL Servers: 172.16.30.10-12
  ├── Port 6032: Admin interface
  ├── Port 6033: MySQL proxy
  └── Query routing 설정
```

**Percona Role**:
```
pxc-1 (Bootstrap):
  ├── Percona XtraDB Cluster 설치
  ├── Bootstrap mode로 시작
  ├── SST user 생성
  ├── Root password 설정
  └── Galera cluster 초기화
  
pxc-2, pxc-3:
  ├── Percona XtraDB Cluster 설치
  ├── pxc-1에 연결
  ├── SST로 데이터 동기화
  └── 클러스터 가입
```

## 6. 연결 방식

### 6-1. HAProxy (DMZ VIP)

**읽기 연결 (Port 3306)**:
```bash
mysql -h172.16.20.30 -P3306 -u root -p
# Round-robin: pxc-1 → pxc-2 → pxc-3
```

**쓰기 연결 (Port 3307)**:
```bash
mysql -h172.16.20.30 -P3307 -u root -p
# Least connections: 가장 적은 연결 노드
```

### 6-2. ProxySQL (DMZ VIP)

**MySQL Proxy (Port 6033)**:
```bash
mysql -h172.16.20.35 -P6033 -u app_user -p
# 쿼리 라우팅: 읽기 → 모든 노드, 쓰기 → writer
```

**Admin Interface (Port 6032)**:
```bash
mysql -h172.16.20.25 -P6032 -uadmin -padmin
# ProxySQL 설정, 모니터링
```

### 6-3. Percona 직접 (Internal - DMZ에서만)

```bash
# DMZ VM에서 실행
ssh kosa@172.16.20.20
mysql -h172.16.30.10 -u root -p

# 외부에서 접근 불가 (pfSense 차단)
```

```bash

  --user=root --password=$PASSWORD \
  --host=172.16.30.10
```

## 9. 문제 해결

### 9-1. Terraform 에러

**Template not found**:
```
Error: Unable to find template 9001

확인:
  ├── qm list | grep 9001
  ├── Template 이름 확인
  └── Template storage 확인 (rbd-storage)
```

**API Token 에러**:
```
Error: Permission denied

확인:
  ├── API Token ID: root@pam!terraform
  ├── Token Secret
  └── Token Permissions: All
```

### 9-2. Ansible 에러

**SSH 연결 실패**:
```
Error: SSH connection refused

확인:
  ├── Cloud-init 완료 여부
  ├── VM IP 확인
  ├── User: kosa, Password: kosa1004
  └── SSH Key
```

**Percona 설치 실패**:
```
Error: Package not found

확인:
  ├── Percona repository
  ├── apt update
  └── Ubuntu 22.04
```

### 9-3. Percona 클러스터 에러

**Split-Brain**:
```
wsrep_cluster_status: Non-Primary

해결:
  1. 모든 MySQL 정지
     systemctl stop mysql
     
  2. Bootstrap
     systemctl start mysql
     
  3. 다른 노드 시작
     systemctl start mysql
```

**노드 가입 실패**:
```
wsrep_local_state_comment: Joining

확인:
  ├── SST user credentials
  ├── Network connectivity
  ├── Firewall (3306, 4444, 4567)
```
